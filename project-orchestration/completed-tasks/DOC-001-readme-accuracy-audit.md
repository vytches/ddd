## What was done

Automated audit of all 20 package READMEs revealed that 17 packages contained
import examples referencing APIs that do not exist in the library source.

### Root cause

READMEs were generated by AI without cross-referencing actual `src/index.ts` exports.
Many examples showed hallucinated class names (`AuthorizationService`,
`RoleBasedAccessControl`, `SagaOrchestrator`, `EventStore`, `ConnectionPoolManager`, etc.)

### Fix

All 17 affected READMEs rewritten from scratch using `src/index.ts` as the
sole source of truth. Key corrections:

- **messaging**: Removed all Saga content (`SagaOrchestrator`, `BaseSagaMiddleware`,
  `InMemorySagaRepository`, `SagaStatus`). Package contains only Outbox pattern.
- **enterprise**: Removed `AuthorizationService`, `RoleBasedAccessControl`,
  `PolicyBasedAccessControl`, `ConnectionPoolManager`, `EventStore`.
- **value-objects**: Removed domain examples (`Address`, `Email`, `Money`,
  `PhoneNumber`) — these are not library exports.
- **cqrs**: Removed `TestCommandBus`, `TestQueryBus`, `Cacheable`, `createMockExecutionContext`.
- **repositories**: Removed `BaseRepository`, `InMemoryStorageAdapter`,
  `UnitOfWork` (concrete class — only interface exists).
- **validation**: Removed `AsyncSpecification` (wrong name), `DomainValidator`,
  `PolicyBuilder`, `Rules`, `ValidationContext`.
- All others: corrected cross-package import paths, removed non-existent helpers.

### Verification

Re-ran automated audit after fix: 0 genuinely fake APIs remaining.
Residual script flags were false positives (cross-package imports correctly
pointing to actual source packages, decorator aliases like `Resilience as ResilienceDecorator`).
